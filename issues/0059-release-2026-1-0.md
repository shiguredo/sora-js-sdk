## 解決方法

1. 「リリースに含めたい open issue」セクションの issue を順次 close する
   - 未磨き上げのものは `/polish-issue` で磨いてから着手する
   - 1 issue = 1 ブランチ = 1 PR = 1 squash merge (`shiguredo-git` の流儀)
2. `CHANGES.md` の `## develop` を `## 2026.1.0` に変更し、`**リリース日**: YYYY-MM-DD` を追記する
   - 記法は `shiguredo-changelog` 規約に従う
   - 直近の 2025.2.0 セクション (`CHANGES.md:100-181`) を参考にする
3. release ブランチ `release/2026.1.0` を切る (前回 2025.2.0 と同じ進行)
4. `package.json` の `version` を `2026.1.0` に bump する
5. release ブランチを `main` および `develop` に merge する
6. tag `2026.1.0` を push し、`npm-publish.yml` の publish ジョブ完走を待つ
   - `verify-version` ジョブが tag 名と `package.json` の version 一致を検証するため、bump 漏れがあれば publish 前に fail する
   - publish 経路は `npm publish --no-git-checks --provenance` (`.github/workflows/npm-publish.yml:103`)
7. GitHub Release を作成し、本リリースの主要変更点を `CHANGES.md` から転記する
8. `develop` に戻り、`CHANGES.md` 先頭に新しい `## develop` セクションを追加する
9. `package.json` を次の canary バージョン (例: `2026.2.0-canary.0`) に bump する
